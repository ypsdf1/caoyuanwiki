# MC草原探险服 Wiki 部署指南

## 🚀 部署到 GitHub Pages（完整步骤）

本文档说明如何将此 GitBook 文档部署到 GitHub Pages。

---

## 第一步：初始化 Git 仓库并推送

打开 **命令提示符（CMD）** 或 **PowerShell**，执行以下命令：

```bash
# 1. 进入项目目录（将此路径替换为你的实际路径）
cd 你解压/存放本项目的目录

# 2. 初始化 git 仓库
git init

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "初始化文档"

# 5. 关联 GitHub 远程仓库
git remote add origin https://github.com/ypsdf1/caoyuanwiki.git

# 6. 推送到 main 分支
git branch -M main
git push -u origin main
```

---

## 第二步：在 GitHub 仓库开启 Pages

1. 打开仓库页面：https://github.com/ypsdf1/caoyuanwiki
2. 点击顶部 **Settings**（设置）
3. 左侧菜单找到 **Pages**
4. 在 **Source** 区域选择：
   - **Deploy from a branch** → 改为 **GitHub Actions**
5. 点击 **Save**（保存）

---

## 第三步：等待自动构建

- 推送代码后，GitHub Actions 会自动触发构建
- 可在仓库的 **Actions** 标签页查看构建进度
- 构建成功后，访问地址为：**https://ypsdf1.github.io/caoyuanwiki/**

## 🔄 后续更新文档

每次修改文档后，只需执行：

```bash
git add .
git commit -m "更新文档内容"
git push
```

GitHub Actions 会自动重新构建并发布。

---

## ❓ 常见问题

### 构建失败怎么办？
1. 在仓库 Actions 页面查看错误日志
2. 最常见原因：`SUMMARY.md` 中引用了不存在的文件路径

### 如何添加新页面？
1. 在 `wikis/` 目录新建 `.md` 文件
2. 在 `SUMMARY.md` 中添加对应链接
3. 推送后自动更新

### 想绑定自定义域名？
1. 在 `Settings > Pages` 中填写自定义域名
2. 在 `wikis/` 同级目录新建 `CNAME` 文件，内容为你的域名
